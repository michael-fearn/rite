## What to build

Create the non-destructive OpenTofu scaffolding that reads Host and VM inventory YAML, generates provider alias HCL from Host YAML, and establishes the static per-Host VM partition pattern required by OpenTofu provider alias limitations. This issue must not create, update, or destroy any Proxmox VM.

## Acceptance criteria

- [x] A committed generator reads `inventory/hosts/*.yaml` as the source of truth for Host provider aliases.
- [x] Generated provider HCL is ignored build output under `tofu/`, not committed source.
- [x] `.gitignore` ignores the generated provider/partition HCL outputs and tofu local state/cache artifacts.
- [x] Generated HCL contains one literal `provider "proxmox"` alias per Host.
- [x] Generated HCL contains one sensitive token variable per Host, e.g. `variable "pve_token_wintermute" { sensitive = true }`.
- [x] Generated HCL contains one literal per-Host VM module or resource-partition block that filters VMs by `placement.host`.
- [x] The per-Host partition pattern does not attempt dynamic provider indexing such as `proxmox[each.value.host]`.
- [x] The root tofu module decodes VM YAML from `inventory/vms/*.yaml`.
- [x] The scaffold preserves one root module and one future fleet state file.
- [x] Tests cover provider alias generation from Host YAML.
- [x] Tests fail clearly when a VM `placement.host` has no generated Host provider coverage.
- [x] Tests or static checks verify all generated token variables are marked sensitive.
- [x] Documentation in this issue or adjacent repo docs explains the generated provider strategy and why pure dynamic HCL aliases are not used.

## Blocked by

None

## Design decisions

- YAML is the source of truth; Host YAML drives provider alias generation.
- Generated provider config is ignored build output.
- The first slice is intentionally non-destructive.
- The core shape is one literal Host alias and one literal Host partition block per Host.
- The scaffold may use a no-op or validation-only VM module until the VM resource bridge issue adds real Proxmox resources.

## Out of scope

- Creating `proxmox_virtual_environment_vm` resources.
- Decrypting PVE API tokens.
- Running `tofu plan`, `tofu apply`, or `tofu destroy` against a real Host.
- VM Prepare, Configure, `vm-up`, or `vm-destroy` workflows.
