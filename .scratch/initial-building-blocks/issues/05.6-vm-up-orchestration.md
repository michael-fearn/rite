## What to build

Replace the `vm-up` stub with the unified operator workflow. `just vm-up vm=<name>` wraps safe scripts that run Prepare, selected-VM tofu plan with explicit approval, selected-VM tofu apply, and Configure.

## Acceptance criteria

- [ ] `just vm-up vm=<name>` invokes a real script such as `scripts/vm-up <name>`.
- [ ] The script rejects undeclared VMs before any mutation.
- [ ] The script runs VM Prepare first.
- [ ] The script runs selected-VM tofu plan through the tofu wrapper.
- [ ] The script requires explicit operator approval after showing the plan and before apply.
- [ ] The script runs selected-VM tofu apply through the tofu wrapper only after approval.
- [ ] The script runs VM Configure after successful apply.
- [ ] Approval safety lives in the script, not only in the just recipe.
- [ ] `just` remains the unified operator interface while scripts remain safe direct debugging entry points.
- [ ] Tests use fake Prepare, fake tofu wrapper, and fake Configure commands to verify order, refusal, approval denial, and approval success.
- [ ] A failed Prepare, plan, apply, or Configure stops the sequence and reports the failed phase.

## Design decisions

- The operator-facing command is `just vm-up vm=<name>`.
- The script owns the safety semantics so direct script invocation is not a footgun.
- The selected-VM apply path avoids accidentally applying every VM in inventory during early rollout.
