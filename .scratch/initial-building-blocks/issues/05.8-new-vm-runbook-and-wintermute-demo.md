## What to build

Write the new VM runbook and perform the real end-to-end demo on `wintermute` once the VM lifecycle slices are implemented. This issue is the acceptance/demo wrapper for the full split.

## Acceptance criteria

- [ ] `runbooks/new-vm.md` documents adding `inventory/vms/<vm>.yaml`.
- [ ] The runbook documents Prepare behavior, including refusal when the VM Sibling SOPS File already exists.
- [ ] The runbook documents that the VM public key is plaintext in VM YAML and the private key is encrypted in the VM Sibling SOPS File.
- [ ] The runbook documents `just vm-up vm=<name>` as the operator path.
- [ ] The runbook documents the tofu plan approval flow.
- [ ] The runbook documents Configure behavior and cloud-init waiting.
- [ ] The runbook documents `just vm-destroy vm=<name>` and its Service Backend refusal.
- [ ] The runbook states destroy removes only the provisioned Proxmox VM.
- [ ] The runbook states destroy leaves `inventory/vms/<vm>.yaml` and `inventory/vms/<vm>.sops.yaml` in the repo.
- [ ] The runbook documents when an AFK agent must stop and alert the maintainer for real Host access, destructive approval, or manual intervention.
- [ ] Demo provisions a test VM end-to-end on `wintermute` through `just vm-up vm=<name>`.
- [ ] Demo destroys the test VM cleanly through `just vm-destroy vm=<name>`.
- [ ] Demo notes include exact command/output if maintainer intervention is needed.
