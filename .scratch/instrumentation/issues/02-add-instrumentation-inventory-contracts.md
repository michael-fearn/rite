# Add Instrumentation inventory contracts

Status: done

## What to build

Add first-class Instrumentation declarations to inventory so VMs can opt out of default VM-level Instrumentation and Services can declare application-specific Telemetry Targets. The completed slice should validate the new inventory shapes and expose query/projection helpers that later workflows can consume without traversing raw YAML.

## Acceptance criteria

- [x] VM inventory accepts `instrumentation.enabled` and treats absent instrumentation as enabled for ordinary declared VMs.
- [x] Service inventory accepts `instrumentation.telemetry_targets` with first-pass `prometheus_metrics` and `http_probe` target types.
- [x] Telemetry Targets require the VM-reachable Published Port and allow only first-pass scheme/path customization.
- [x] Cross-file validation rejects Telemetry Targets that reference unreachable or undeclared published ports.
- [x] Inventory graph or intent helpers expose instrumented ordinary VMs and Service Telemetry Targets for downstream workflows.
