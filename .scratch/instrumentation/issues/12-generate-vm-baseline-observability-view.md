# Generate VM baseline Observability View

Status: done

## What to build

Generate a first-pass `vm_baseline` Observability View from enabled VM-level Instrumentation. The view should give the Operator a baseline VM-oriented dashboard over the node exporter and VM log signals already collected by the Observability Service, without requiring per-VM dashboard declarations.

## Acceptance criteria

- [x] Enabled VM-level Instrumentation causes VMs to appear in generated baseline VM Observability Views.
- [x] VMs with VM-level Instrumentation opted out do not appear in generated baseline VM Observability Views.
- [x] Generated VM baseline view identity is stable and derived from the VM identity and view kind.
- [x] Generated dashboard JSON references the Prometheus and Loki data sources through provisioning-safe identifiers or variables.
- [x] Focused tests prove dashboard generation for included VMs, opted-out VMs, stable identity, and datasource references.
