# Inventory Entity Graph networking/query basics

## What to build

Add the first Inventory Entity Graph tracer bullet: a read-only query Module over loaded Inventory that answers common networking and cross-Entity lookup questions without requiring callers to traverse raw per-Entity YAML shape.

The Inventory Entity Graph should preserve flat Inventory YAML as the source of truth. It should be constructed from the existing loaded Inventory model, expose an explicit public Interface that accepts Entity names, and return normalized string IPv4 facts for callers that render Caddy routes, dnsmasq records, Ansible vars, OpenTofu target planning, or operator messages.

For ordinary absence, query methods should return `None`. For invalid or ambiguous declared facts, they should raise a domain-specific Inventory Entity Graph error so workflow callers get clear failures and validation modules can adapt errors into validation diagnostics.

- [ ] Public graph methods accept Entity names rather than raw Entity dictionaries.
