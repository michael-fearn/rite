# Migrate Service-layer Acceptance Test to Inventory Entity Graph intent

## Parent

.scratch/inventory-entity-graph/issues/03-workflow-intent-queries.md

## What to build

Wire the Service-layer Acceptance Test workflow into the Inventory Entity Graph Acceptance Policy intent query so the workflow no longer assembles selected Host, Template, NAS Endpoint, generated Operational VM, Ephemeral Dataset, and Host bridge facts through raw Inventory traversal.

The workflow should continue to prove the Service-layer contract end to end: generate temporary Inventory artifacts, encrypt the generated Service Sibling SOPS File, reconcile the Ephemeral Dataset and Derived NFS Share, provision Acceptance VMs, deploy generated Quadlet and Native Services, verify Service data flow and secrets, reload the TrueNAS NFS service, verify post-reload behavior, and clean up generated resources. The Inventory Entity Graph should supply read-only intent facts only.
