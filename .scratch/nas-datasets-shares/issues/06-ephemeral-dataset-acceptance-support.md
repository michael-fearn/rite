# Ephemeral Dataset acceptance support

Status: done
Category: enhancement

## What to build

Add Acceptance Test support for Ephemeral Datasets so live NAS behavior can be proven without risking adopted data. Ephemeral Datasets may be created and destroyed only inside acceptance-test inventory and workflows, and can be used to verify Dataset validation, derived Share creation, VM Mount access, and Share-backed Service volume behavior end to end.

## Acceptance criteria

- [ ] Acceptance-test inventory can declare `lifecycle: ephemeral` Datasets without weakening ordinary fleet validation.
- [ ] Acceptance workflow creates an Ephemeral Dataset, runs NAS Reconcile, configures a disposable Operational VM Mount, and verifies read/write/delete access according to declared Access Policy.
- [ ] Acceptance workflow verifies `source: /` and safe relative Share-backed Volume subpath behavior.
- [ ] Acceptance workflow destroys Ephemeral Datasets and fortress-owned Shares it created.
- [ ] Failure handling avoids deleting ordinary adopted Datasets or unmanaged Shares.

## Agent Brief

**Category:** enhancement
**Summary:** Add Acceptance Test support for Ephemeral Datasets that proves NAS Dataset, Share, Mount, and Share-backed Volume behavior against live infrastructure without endangering adopted data.

**Current behavior:**
Ordinary Inventory can declare Datasets, VM Mounts, and Service Share-backed Volumes. Cross-file validation already rejects `lifecycle: ephemeral` Datasets unless Acceptance Test inventory explicitly opts in, and it already validates Share-backed Volume `source` values so `/` means the Mount root while non-root sources must be safe relative subpaths. NAS Reconcile validates adopted Datasets and can reconcile fortress-owned derived NFS Shares without mutating unmanaged Shares.

There is not yet an Acceptance Test workflow that declares disposable Ephemeral Datasets, creates them on TrueNAS, runs NAS Reconcile against them, configures a disposable Operational VM Mount, verifies VM read/write/delete access, verifies Share-backed Volume consumption, and then destroys only the Ephemeral Datasets and fortress-owned Shares it created.

**Desired behavior:**
Acceptance Test inventory and workflows should be able to create, exercise, and destroy Ephemeral Datasets in a constrained live-test context. Ephemeral Dataset support must be opt-in and must not weaken ordinary fleet validation or make adopted Dataset creation/deletion possible.

The Acceptance Test flow should create an Ephemeral Dataset, reconcile the derived fortress-owned Share, configure a disposable Operational VM Mount, and verify read/write/delete behavior according to the declared Access Policy. It should also verify Service Share-backed Volume behavior for both `source: /` and safe relative subpaths. Cleanup must be conservative: only Ephemeral Datasets created by the Acceptance Test and fortress-owned Shares created for the test may be destroyed. Failure handling should prefer leaving suspicious resources behind with a clear report over risking deletion of ordinary adopted Datasets or unmanaged Shares.

**Key interfaces:**
- Inventory validation mode for Acceptance Tests — should continue to require explicit opt-in for `lifecycle: ephemeral`, while ordinary fleet validation continues to reject Ephemeral Datasets.
- Dataset lifecycle model — should distinguish adopted Datasets from Ephemeral Datasets strongly enough that create/destroy operations can only target Ephemeral Datasets in Acceptance Test inventory.
- NAS Reconcile and TrueNAS client behavior — should support creating and destroying Ephemeral Datasets for Acceptance Tests while keeping ordinary adopted Dataset validation read-only and protected.
- Fortress-owned Share reconciliation — should continue to create, update, and destroy only Shares carrying the fortress ownership marker.
- Acceptance Test orchestration — should provision or use a disposable Operational VM, run NAS Reconcile, run VM Configure for Mount setup, exercise VM Mount access, exercise Share-backed Volume access, and clean up test-owned resources.
- Share-backed Volume validation and deployment behavior — should prove that `source: /` binds the Mount root and safe relative subpaths are handled as expected.

**Acceptance criteria:**
- [ ] Ordinary fleet validation still rejects `lifecycle: ephemeral` Datasets unless Acceptance Test inventory explicitly opts in.
- [ ] Acceptance Test inventory can declare an Ephemeral Dataset and a disposable Operational VM Mount without allowing ordinary Inventory to create or destroy Datasets.
- [ ] The Acceptance Test workflow creates an Ephemeral Dataset, runs NAS Reconcile, configures the disposable Operational VM Mount, and verifies read/write/delete access for a read-write Access Policy.
- [ ] The Acceptance Test workflow verifies read-only Access Policy behavior does not allow writes through the VM Mount.
- [ ] The Acceptance Test workflow verifies a Share-backed Volume with `source: /` binds the root of the Mount.
- [ ] The Acceptance Test workflow verifies safe relative Share-backed Volume subpaths are resolved under the Mount and are rejected or reported clearly when missing, according to the existing Service deployment contract.
- [ ] Cleanup destroys only Ephemeral Datasets created by the Acceptance Test and fortress-owned Shares created for the test.
- [ ] Cleanup refuses to delete adopted Datasets, unmanaged Shares, or resources that cannot be proven to belong to the Acceptance Test.
- [ ] Failure handling reports leftover test-owned resources clearly and does not roll back unrelated NAS or VM state.
- [ ] Automated tests cover the acceptance-inventory opt-in, Ephemeral Dataset create/destroy gating, Share cleanup gating, and Share-backed Volume `source: /` plus safe-relative-subpath behavior.

**Out of scope:**
- Creating, deleting, repairing, or adopting ordinary adopted Datasets.
- Mutating or deleting unmanaged TrueNAS Shares.
- Replacing the existing fortress-owned Share marker model.
- Running NAS Reconcile implicitly from ordinary Service deployment.
- General-purpose live infrastructure CI beyond the specific Ephemeral Dataset Acceptance Test workflow.
