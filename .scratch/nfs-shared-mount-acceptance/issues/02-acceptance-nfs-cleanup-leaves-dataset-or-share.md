## Agent Brief

**Category:** bug
**Summary:** Make Acceptance Test cleanup actually remove and verify removal of Ephemeral Datasets and Derived NFS Shares after live runs.

**Current behavior:**
Live Acceptance Test runs can leave both the acceptance-owned Ephemeral Dataset and its fortress-owned Derived NFS Share behind after the test is accepted. The focused NFS shared-mount workflow tests pass, and the workflow asserts that cleanup output contains expected delete actions, but that automated coverage uses test doubles and does not prove live TrueNAS postconditions.

This has been observed for the `acceptance-nfs-demo` dataset and its derived read-write NFS Share. The same cleanup contract also matters for other live Acceptance Tests that create Ephemeral Datasets and fortress-owned Shares.

**Desired behavior:**
After a successful default Acceptance Test run, no generated VMs, generated VM inventory artifacts, acceptance Ephemeral Dataset, or matching fortress-owned Derived NFS Share should remain. Cleanup should verify live postconditions, not only that a plan or API operation was attempted.

If cleanup cannot prove the Dataset and Share were removed, the workflow should fail loudly and report exactly which resource remains. If cleanup cannot prove that a resource is acceptance-owned and safe to delete, it should preserve the resource and report the blocker. Cleanup must remain conservative and must not delete adopted Datasets, unmanaged Shares, unrelated fortress-owned Shares, or any Dataset/Share missing the expected fortress ownership marker.

**Key interfaces:**

- Acceptance Test cleanup contract — default cleanup should destroy generated VM clients first, then remove only acceptance-owned NAS resources, then verify those NAS resources are absent.
- NAS Reconcile acceptance destroy mode — should delete marked Ephemeral Datasets and matching fortress-owned Shares, report blockers for unmarked or unmanaged overlaps, and expose enough result data for workflows to distinguish attempted deletion from verified absence.
- TrueNAS live reality reading — should refresh reality after destructive cleanup, including Dataset and NFS Share markers, so postconditions are based on current NAS state rather than stale planning input.
- Workflow failure reporting — if the test fails and cleanup also fails, the operator should see both the original failure and the cleanup/postcondition failure.
- Automated tests and test doubles — should cover live-style postcondition failures where delete actions are emitted but the Dataset or Share still appears in refreshed reality.

**Acceptance criteria:**

- [ ] A successful default NFS shared-mount Acceptance Test run removes generated VMs and generated VM inventory artifacts.
- [ ] The same run removes the marked `acceptance-nfs-demo` Ephemeral Dataset from live TrueNAS reality.
- [ ] The same run removes the matching fortress-owned read-write Derived NFS Share from live TrueNAS reality.
- [ ] Cleanup verifies absence after deletion; a delete action in the cleanup output is not treated as sufficient proof by itself.
- [ ] If the Dataset remains after cleanup, the workflow exits non-zero and reports the Dataset name/path clearly.
- [ ] If the Share remains after cleanup, the workflow exits non-zero and reports the Share name/path clearly.
- [ ] Cleanup preserves and reports unmarked Ephemeral Datasets, unmanaged Shares, unrelated fortress-owned Shares, and adopted Datasets.
- [ ] If cleanup fails after an earlier workflow failure, the operator sees both the original failure and the cleanup/postcondition failure.
- [ ] Automated tests cover stale-success behavior where cleanup reports delete actions but refreshed reality still contains the Dataset, the Share, or both.
- [ ] Automated tests cover the conservative preservation cases so the fix cannot broaden deletion beyond acceptance-owned resources.

**Out of scope:**

- Changing the Dataset lifecycle model.
- Changing ordinary NAS Reconcile behavior for adopted Datasets.
- Deleting unmanaged or unmarked NAS resources as a convenience cleanup path.
- Broad live infrastructure CI beyond the Acceptance Test cleanup paths that create Ephemeral Datasets and Derived NFS Shares.
