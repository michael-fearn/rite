## What to build

Migrate **VM Lifecycle Convergence** to build an importable **Operator Workflow Plan** and execute it through the **Operator Workflow Runner**.

The operator-facing behavior of `scripts/vm-up` should remain the same: Prepare, selected-VM OpenTofu plan, exact confirmation unless auto-confirmed, selected-VM OpenTofu apply, then Configure. The implementation should move workflow planning into an importable module so tests can inspect the plan and exercise execution through fake or controlled runner behavior.

## Acceptance criteria

- [ ] An importable VM Lifecycle Convergence plan builder creates the ordered plan for one declared VM.
- [ ] The plan contains Prepare, selected-VM OpenTofu plan, a distinct confirmation gate, selected-VM OpenTofu apply, and Configure in the current order.
- [ ] The confirmation gate requires the same exact approval phrase currently required by `scripts/vm-up`.
- [ ] Auto-confirm bypasses the confirmation gate without prompting.
- [ ] Selected-VM OpenTofu target construction remains workflow-specific and does not move into the runner.
- [ ] `scripts/vm-up` becomes a thin entrypoint that parses args, validates or resolves the VM, builds the plan, runs it, renders the result, and exits with the result status.
- [ ] Current operator-visible success, denial, missing VM, failed phase, unknown flag, and missing command behavior remains covered by tests.
- [ ] Tests assert the VM Lifecycle Convergence plan shape without spawning downstream commands for every behavior.
- [ ] Tests still cover the script entrypoint wiring at least once end to end.
