## What to build

Migrate **Service Launch** to build an importable **Operator Workflow Plan** and execute it through the **Operator Workflow Runner**.

The operator-facing behavior of `scripts/service-launch` should remain the same: resolve one declared Service, run Backend VM readiness through VM Lifecycle Convergence, run Service Deploy, and run Ingress Regeneration only when the Service declares Ingress.

## Acceptance criteria

- [ ] An importable Service Launch plan builder creates the ordered plan for one declared Service.
- [ ] The plan resolves the Service's Backend VM through existing Inventory domain queries.
- [ ] The plan runs VM Lifecycle Convergence before Service Deploy.
- [ ] The plan passes auto-confirm through only to VM Lifecycle Convergence.
- [ ] The plan runs Ingress Regeneration only when the Service declares Ingress.
- [ ] `scripts/service-launch` becomes a thin entrypoint that parses args, validates or resolves the Service, builds the plan, runs it, renders the result, and exits with the result status.
- [ ] Missing Service, missing Backend, missing Backend VM, downstream phase failure, Ingress-enabled, non-Ingress, and singular-Service scope behavior remains covered by tests.
- [ ] Tests assert the Service Launch plan shape without spawning downstream commands for every behavior.
- [ ] Tests still cover the script entrypoint wiring at least once end to end.
