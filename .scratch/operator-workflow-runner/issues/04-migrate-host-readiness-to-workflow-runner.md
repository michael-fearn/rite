## What to build

Migrate **Host Readiness** to build an importable **Operator Workflow Plan** and execute it through the **Operator Workflow Runner**.

This slice should include the hard Host Readiness behavior that shaped the runner: streaming phase output with prefixes, captured diagnostic tails, stop versus continue failure policy, Acceptance Test matrix continuation, and final structured summaries.

## Acceptance criteria

- [ ] An importable Host Readiness plan builder creates the ordered plan for one selected Host, selected endpoint scope, auto-confirm policy, and keep-on-fail policy.
- [ ] The plan represents Bootstrap satisfaction or Bootstrap execution before Host reachability proof.
- [ ] The plan runs Host Configure, Template Build, Template Verification for every declared Template, and selected Acceptance Tests in the current order.
- [ ] The plan expands `endpoint=all` into all declared NAS Endpoints and preserves single-endpoint scope when requested.
- [ ] The plan uses streaming command phases for long-running Host Readiness phases and prefixes output consistently through the runner.
- [ ] Template and prerequisite failures stop immediately.
- [ ] Acceptance Test failures continue across independent Template x NAS Endpoint cells when `keep_on_fail=false`.
- [ ] Acceptance Test failures stop when `keep_on_fail=true` would preserve artifacts that can collide with later cells.
- [ ] The script exits nonzero if any phase or Acceptance Test cell fails.
- [ ] `scripts/host-up` becomes a thin entrypoint that parses args, resolves the Host Readiness plan, runs it, renders the final summary, and exits with the result status.
- [ ] Existing command signature, just recipe wiring, Bootstrap missing/satisfied/ambiguous paths, Host reachability proof, Configure tags, no-Template failure, Template ordering, endpoint resolution, Acceptance Test matrix expansion, auto-confirm, keep-on-fail, failure aggregation, and summary output remain covered by tests.
- [ ] Tests assert Host Readiness plan shape and runner failure policy without spawning downstream commands for every behavior.
