# Quadlet Service Secret facts through Service Runtime Intent

Status: ready-for-agent
Category: enhancement
Date: 2026-05-19

## Parent

`.scratch/service-runtime-intent/PRD.md`

## What to build

Move Quadlet Service Secret reference and wiring meaning into Service Runtime Intent. Fleet-wide analysis should expose Service Secret facts for Quadlet containers, including the secret key, service-scoped Podman secret name, environment variable name, SOPS extraction path, and whether the container receives a secret file path or the Podman secret name.

Inventory validation should adapt Service Runtime Intent diagnostics for invalid Service Secret references and invalid environment wiring. Service Deploy should use the facts to build the same secret installation vars and preflight required keys. Quadlet rendering should use the facts for `Secret=` and `Environment=` lines while preserving rendered output.

Service Runtime Intent must not decrypt SOPS, hold plaintext secret values, or become a deploy-var renderer.

## Acceptance criteria

- [ ] Service Runtime Intent exposes Quadlet Service Secret facts identifying Service, container, secret index, secret key, service-scoped Podman secret name, environment variable name, SOPS extraction path, and env value mode.
- [ ] Service Runtime Intent diagnoses Service Secret references that do not use `secrets.<name>`.
- [ ] Service Runtime Intent diagnoses Service Secret file-path wiring whose environment variable does not end in `_FILE`.
- [ ] `env_value: secret_name` remains supported and is represented as an explicit runtime fact.
- [ ] Inventory validation adapts Service Secret diagnostics while preserving current validation codes and operator-facing messages wherever practical.
- [ ] Service Deploy still produces the same Service Secret installation variables and preflight required secret keys.
- [ ] Quadlet rendering still produces unchanged Podman `Secret=` lines and Service Secret environment wiring.
- [ ] Direct Service Runtime Intent tests cover service-scoped Podman names, `secrets.<name>` extraction paths, default `_FILE` path wiring, `env_value: secret_name`, invalid references, invalid env wiring, and partial facts with diagnostics.
