## Agent Brief

**Category:** enhancement
**Summary:** Add endpoint-explicit live NAS Reconcile entrypoints that decrypt the NAS Reconcile Credential from the NAS Endpoint Sibling SOPS File into the child process environment.

**Current behavior:**
NAS Reconcile is currently fixture-backed: the command requires a `--reality-json` file, uses offline TrueNAS reality data, and remains credential-free. The command reports NAS Endpoint connection settings without printing secret values, but it does not yet support a live endpoint mode that loads TrueNAS reality through the endpoint's Management Address.

**Desired behavior:**
Live NAS Reconcile should accept an explicit NAS Endpoint name, for example `--live truenas`, for both plan and apply workflows. Before any network access, the workflow should require the named NAS Endpoint Inventory file and its Sibling SOPS File, read the endpoint's `api_token_env`, decrypt `api_credentials.reconcile.value` from the Sibling SOPS File, and export that value only into the child process environment variable named by `api_token_env`.

Fixture-backed `--reality-json` mode should stay endpoint-implicit, credential-free, and usable in tests without any NAS Endpoint Sibling SOPS File. Operator errors for missing endpoint Inventory, missing Sibling SOPS File, missing `api_token_env`, failed SOPS extraction, or failed live credential preparation should be clear and must not print the credential value.

**Key interfaces:**
- `scripts/nas-reconcile-plan` command-line interface - should support mutually exclusive fixture-backed and endpoint-explicit live modes while preserving existing apply and confirmation flags.
- NAS Endpoint Inventory shape - `name`, `management_address`, `share_address`, and `api_token_env` remain non-secret configuration; plaintext inline API token fields should not be introduced.
- NAS Endpoint Sibling SOPS File shape - ordinary NAS Reconcile Credential lives at `api_credentials.reconcile.value`.
- Live TrueNAS reality loading boundary - should receive the decrypted credential only through the process-local environment variable declared by the endpoint, and should not leak the value into plan output, errors, logs, or temporary files.
- `just` command surface - should expose endpoint-explicit live plan and apply tasks in addition to existing fixture-backed tasks.

**Acceptance criteria:**
- [ ] `scripts/nas-reconcile-plan --live truenas` requires `inventory/nas/truenas.yaml` and `inventory/nas/truenas.sops.yaml` before attempting network access.
- [ ] The live workflow decrypts `api_credentials.reconcile.value` from `inventory/nas/truenas.sops.yaml` and exports it only to the child process environment named by `api_token_env`.
- [ ] Missing endpoint, missing Sibling SOPS File, missing `api_token_env`, and failed SOPS extraction produce clear operator errors without printing the credential value.
- [ ] Existing `--reality-json` plan/apply tests continue to pass without any NAS Endpoint Sibling SOPS File.
- [ ] `just` exposes endpoint-explicit live plan/apply tasks.
- [ ] Live plan/apply output may report the Credential Source, endpoint name, and non-secret connection settings, but never the NAS Reconcile Credential value.

**Out of scope:**
- Replacing fixture-backed `--reality-json` mode.
- Storing plaintext API token values in NAS Endpoint Inventory.
- Creating, updating, or deleting ordinary Datasets.
- Implementing the deprecated TrueNAS REST API surface.
- Adding Acceptance NAS Credential support for Ephemeral Dataset tests.
