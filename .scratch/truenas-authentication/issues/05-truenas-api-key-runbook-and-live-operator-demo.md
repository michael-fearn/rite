## Agent Brief

**Category:** enhancement
**Summary:** Document the NAS Credential Ceremony and a live read-only NAS Reconcile operator demo.

**Current behavior:**
Live NAS Reconcile now has the foundations needed for endpoint-explicit TrueNAS planning: the operator can target a NAS Endpoint, the workflow uses the NAS Endpoint's Management Address, and the ordinary NAS Reconcile Credential is loaded from the endpoint's Sibling SOPS File at `api_credentials.reconcile.value`.

ADR 0019 establishes that ordinary NAS Reconcile credentials may inspect Dataset metadata and manage Fortress-owned NFS Shares, but must not create, update, or delete ordinary Datasets. It also establishes that Acceptance Tests which need Ephemeral Dataset mutation use a separate Acceptance NAS Credential under `api_credentials.acceptance.value`.

The missing piece is operator-facing documentation. There is no runbook that tells an Operator how to perform the NAS Credential Ceremony, store both credentials in the NAS Endpoint Sibling SOPS File, and demonstrate a live read-only plan against TrueNAS without mutating ordinary data.

**Desired behavior:**
The documentation should include a runbook for creating both TrueNAS-side API keys in one operator ceremony:

- `fortress-nas-reconcile`, intended for ordinary NAS Reconcile with Dataset-read and NFS-Share-manage capability.
- `fortress-acceptance-ephemeral-datasets`, intended only for acceptance workflows that create or destroy Ephemeral Datasets.

The runbook should tell the Operator where each credential belongs in `inventory/nas/<endpoint>.sops.yaml`: the ordinary NAS Reconcile Credential at `api_credentials.reconcile.value`, and the Acceptance NAS Credential at `api_credentials.acceptance.value`.

The documentation should also include a live operator demo checklist that proves endpoint-explicit live NAS Reconcile can run preflight and produce a read-only plan with `--live truenas` without mutating TrueNAS. The checklist should include expected failure modes for missing SOPS material and insufficient TrueNAS privilege, and it should make clear that routine NAS Reconcile credentials must not mutate ordinary Datasets.

Documentation should link back to ADR 0019 and use the established glossary terms: NAS Endpoint, Management Address, Share Address, NAS Reconcile Credential, NAS Credential Ceremony, Acceptance NAS Credential, and Ephemeral Dataset.

**Key interfaces:**
- NAS Credential Ceremony — the manual operator process for creating both NAS API credentials outside fortress before storing them in the NAS Endpoint's Sibling SOPS File.
- NAS Endpoint Sibling SOPS File shape — `inventory/nas/<endpoint>.sops.yaml` should contain `api_credentials.reconcile.value` for ordinary NAS Reconcile and `api_credentials.acceptance.value` for acceptance-only Ephemeral Dataset mutation.
- Live NAS Reconcile command surface — the operator demo should use endpoint-explicit live mode such as `scripts/nas-reconcile-plan --live truenas`.
- ADR 0019 — the runbook should cite the authentication model, credential source behavior, and ordinary Dataset mutation boundary from the ADR.
