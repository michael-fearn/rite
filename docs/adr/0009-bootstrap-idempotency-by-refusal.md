# Bootstrap idempotency by refusal, not silent skip

`host-bootstrap.yml` refuses to run if the Host Sibling SOPS File already exists, while `vm-prepare.yml` refuses to run if the VM Sibling SOPS File already contains VM SSH key material. Bootstrap and Prepare are the only steps that mint irreversible SSH material; a silent skip would either overwrite a valid key or no-op past a real divergence. VM Prepare may merge generated SSH key material into a pre-existing VM Sibling SOPS File that contains non-SSH credentials, such as a Tailnet Auth Key, because those credentials must exist before Configure but do not mean the VM SSH key has already been minted.
