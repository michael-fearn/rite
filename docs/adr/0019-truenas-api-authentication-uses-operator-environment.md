# TrueNAS API authentication uses SOPS-backed operator environment

NAS Reconcile authenticates to a NAS Endpoint's management API with a NAS Reconcile Credential stored in the endpoint's Sibling SOPS File and exported by the operator workflow into the endpoint-declared environment variable. NAS Endpoints are first-class Inventory Entities, distinguish the Management Address used for API access from the Share Address consumed by VMs, and live workflows fetch TrueNAS reality inside the workflow before planning or applying changes; fixture-backed NAS Reconcile Plans remain credential-free. Live NAS Reconcile uses the official `truenas_api_client` package for the current TrueNAS SCALE API client/WebSocket API surface rather than implementing the protocol itself or using the deprecated REST API. This keeps TrueNAS credentials consistent with the rest of fortress secrets while still giving the API client a narrow runtime interface.

Consequences: NAS Reconcile may report the Credential Source but never the credential value, inline `api_token` values in plaintext Inventory should be schema-invalid, and TrueNAS write failures are handled by forward retry through the next reconcile run rather than rollback. The ordinary NAS Reconcile Credential can read Dataset metadata and manage NFS Shares, but it must not create, update, or delete ordinary Datasets; Acceptance Tests that need Ephemeral Dataset mutation use a separate Acceptance NAS Credential. Live workflows perform a non-mutating credential preflight before planning or applying: missing encrypted credential material, failed environment export, connection failure, and missing read capabilities fail before reconciliation logic runs, while write capability is checked only if TrueNAS exposes a safe non-mutating permission check.

The ordinary credential path is `api_credentials.reconcile.value` in `inventory/nas/<endpoint>.sops.yaml`; acceptance-only credentials use a separate purpose under `api_credentials`. The operator workflow decrypts the SOPS value directly into the child process environment variable named by `api_token_env`; it does not write a temporary token file.

Live NAS Reconcile commands must name the NAS Endpoint explicitly, for example `--live truenas`; both live plan and live apply require the endpoint's Sibling SOPS File because both read TrueNAS through the management API. Fixture-backed commands remain endpoint-implicit, credential-free, and available for offline planning and tests.
