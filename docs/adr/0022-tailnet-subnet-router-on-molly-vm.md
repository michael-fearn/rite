# Tailnet subnet router runs as a VM on molly

The immediate remote-operator path uses a small ordinary VM on the `molly` Host as the Tailnet Subnet Router for the Hosted Tailnet, attached to the Trusted VLAN because it extends trusted Operator reachability rather than serving ordinary infrastructure clients. It is routing-only, not a Remote Operator Workstation, and enrolls with a Tailnet Auth Key stored in its Sibling SOPS File so rebuilds are declared rather than interactive. Its setup is VM-level configuration rather than a Service, because Tailscale route advertisement and forwarding are host-networking concerns rather than application deployment. A standalone device would be a stronger recovery boundary, but the VM is faster to create with existing fortress workflows, keeps Tailscale out of individual Service VMs, and contains tailnet routing away from the Proxmox host OS. During early bring-up it advertises all fortress VLAN routes so the Operator is not stranded away from home by a missing route; the trade-off is explicit: remote reachability now depends on `molly` and its VM lifecycle being healthy, and tailnet route scope must be tightened later when the baseline is stable.
