# Instrumentation convergence is explicit

Fortress treats Instrumentation as declared observability wiring owned by the observed VM or Service, while the Observability VM consumes those declarations to collect telemetry. VM-level Instrumentation is applied by VM Configure and a dedicated Instrumentation Convergence workflow refreshes ordinary VMs and the Observability Service; Service Deploy remains scoped to the named Service and does not implicitly refresh observability. Higher-level workflows such as Service Launch and Service Group Launch may refresh the Observability VM when launched Services change Instrumentation, preserving narrow low-level command boundaries while still giving operators a single explicit path to apply observability changes across existing VMs and Services.
