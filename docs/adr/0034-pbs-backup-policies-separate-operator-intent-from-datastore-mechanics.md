# PBS backup policies separate operator intent from datastore mechanics

Backup Targets explicitly opt into PBS with `backup.enabled` and select a named Backup Policy, defaulting to `default` when no policy is named. Backup Policy definitions live in fleet-level inventory and own schedule plus retention; the initial committed inventory contains only `default`, a daily Backup Run to the Primary Datastore with daily, weekly, and monthly retention. Each Backup Target gets its own PVE-side Backup Job, with the job schedule deterministically staggered from the policy time by Backup Target identity. Policies are modeled as named entries so future policies can be added without changing VM inventory shape. This keeps VM inventory focused on backup intent while allowing cadence, retention, and datastore mechanics to evolve centrally instead of becoming per-VM configuration.
